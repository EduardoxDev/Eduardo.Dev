# Building a Container Runtime from Scratch in Go

When I started working on [Mini Runtime Container](https://github.com/EduardoxDev/Mini-Runtime-Container), my goal was simple: understand exactly what Docker does when you run `docker run`. Not at a high level — at the syscall level.

The result taught me more about Linux internals than any book.

## What Is a Container, Really?

A container is just a process with restricted visibility. There's no magic — just Linux kernel features that have existed for years:

- **Namespaces** — limit what a process can _see_ (other processes, network, filesystem)
- **cgroups** — limit what a process can _use_ (CPU, memory, I/O)
- **pivot_root** — give the process its own root filesystem

That's it. Docker, containerd, Podman — they're all orchestration layers on top of these primitives.

## Namespaces: Restricting Visibility

Linux has 8 namespace types. The most important for containers:

```go
const isolationFlags = syscall.CLONE_NEWUTS |
    syscall.CLONE_NEWPID |
    syscall.CLONE_NEWNS  |
    syscall.CLONE_NEWNET

cmd := exec.Command(args[0], args[1:]...)
cmd.SysProcAttr = &syscall.SysProcAttr{
    Cloneflags: isolationFlags,
}
```

With `CLONE_NEWPID`, the container process thinks it's PID 1. Kill it, and the kernel cleans up all its children automatically.

## Setting Up the Root Filesystem

```go
func pivotRoot(newRoot string) error {
    if err := syscall.Mount(newRoot, newRoot, "", syscall.MS_BIND|syscall.MS_REC, ""); err != nil {
        return fmt.Errorf("bind mount: %w", err)
    }

    putOld := filepath.Join(newRoot, ".put_old")
    if err := os.MkdirAll(putOld, 0700); err != nil {
        return err
    }

    if err := syscall.PivotRoot(newRoot, putOld); err != nil {
        return fmt.Errorf("pivot_root: %w", err)
    }

    os.Chdir("/")
    syscall.Unmount("/.put_old", syscall.MNT_DETACH)
    return os.Remove("/.put_old")
}
```

## Resource Limiting with cgroups v2

```go
func createCgroup(name string, memLimitMB int) error {
    cgPath := filepath.Join("/sys/fs/cgroup", name)

    if err := os.MkdirAll(cgPath, 0755); err != nil {
        return err
    }

    memLimit := fmt.Sprintf("%d", memLimitMB*1024*1024)
    if err := os.WriteFile(filepath.Join(cgPath, "memory.max"), []byte(memLimit), 0644); err != nil {
        return err
    }

    pid := fmt.Sprintf("%d", os.Getpid())
    return os.WriteFile(filepath.Join(cgPath, "cgroup.procs"), []byte(pid), 0644)
}
```

## What I Learned

1. **Container isolation is shallow** — a misconfigured namespace flag and you're back on the host
2. **cgroups v1 vs v2 are completely different** — most tutorials use v1, but modern systems use v2
3. **Network namespaces require veth pairs** — getting networking working is a project in itself
4. **pivot_root requires specific mount propagation** — this took two days to debug

The full implementation is on [GitHub](https://github.com/EduardoxDev/Mini-Runtime-Container). ~400 lines of Go that does something genuinely interesting.
